# PROJECT BRAIN — THE_WEBSITE (Portfolio)

## What This Project Is
A personal portfolio website built with React + Vite + TypeScript.
Uses Google Gemini AI (@google/genai) on the backend and Motion for animations.
Has an admin panel, project detail views, and an Express backend server.

---

## Tech Stack
| Layer | Tech |
|-------|------|
| Frontend | React 19, TypeScript, Vite 6 |
| Styling | Tailwind CSS v4 |
| Animations | Motion (Framer Motion successor) |
| Icons | Lucide React |
| Backend | Express.js (server.js) |
| AI | Google Gemini (@google/genai v2.4) |
| Runtime | tsx (for running TS server) |

---

## Folder Structure
```
THE_WEBSITE/
├── assets/           → Static assets (images, icons, etc.)
├── data/             → JSON data files (projects, content, etc.)
├── public/           → Public files served as-is (favicon, etc.)
├── src/
│   ├── components/   → All React UI components
│   │   ├── AdminPanel.tsx        → Admin interface to manage portfolio content
│   │   ├── Header.tsx            → Site header / navigation
│   │   ├── ProjectDetailView.tsx → Full view of a single project
│   │   └── ServiceInnerView.tsx  → Detail view for a service/skill
│   ├── server/       → Backend server code
│   │   ├── api.ts    → API route handlers (Gemini AI calls, data endpoints)
│   │   └── db.ts     → Data layer / file-based database logic
│   ├── App.tsx       → Root component, routing logic
│   ├── main.tsx      → App entry point (renders App into DOM)
│   ├── index.css     → Global styles + Tailwind imports
│   └── types.ts      → Shared TypeScript types/interfaces
├── .env.example      → Template for environment variables
├── index.html        → HTML shell (Vite entry point)
├── metadata.json     → Site metadata (title, description, etc.)
├── server.js         → Compiled/built Express backend (auto-generated)
├── vite.config.ts    → Vite configuration
├── tsconfig.json     → TypeScript configuration
└── package.json      → Dependencies and scripts
```

---

## Key Files — What Each Does
| File | Role |
|------|------|
| `src/App.tsx` | Main app shell, handles page/view routing |
| `src/main.tsx` | Mounts React app to `index.html` |
| `src/types.ts` | All shared TypeScript types — check here before creating new ones |
| `src/index.css` | Global CSS + Tailwind v4 setup |
| `src/components/Header.tsx` | Top navigation bar |
| `src/components/AdminPanel.tsx` | Admin UI for editing portfolio content |
| `src/components/ProjectDetailView.tsx` | Renders a single project's full page |
| `src/components/ServiceInnerView.tsx` | Renders a single service/skill detail |
| `src/server/api.ts` | All backend API routes (Gemini AI + data) |
| `src/server/db.ts` | Reads/writes portfolio data (projects, services) |
| `metadata.json` | Site-level info: name, tagline, social links |
| `data/` | Source of truth for portfolio content (projects, services) |

---

## Environment Variables (from .env.example)
- `GEMINI_API_KEY` → Google Gemini API key (required for AI features)
- Copy `.env.example` → `.env` and fill in before running

## npm Scripts
| Command | What it does |
|---------|-------------|
| `npm run dev` | Start frontend dev server at http://localhost:3000 |
| `npm start` | Start Express backend server |
| `npm run build` | Build for production |
| `npm run lint` | TypeScript type check |
| `npm run clean` | Delete dist and server.js |

## Architecture Notes
- Frontend and backend are **separate servers** in dev mode
- Frontend: Vite on port 3000
- Backend: Express (port likely 8080 or defined in server/api.ts)
- Gemini AI calls go through the Express backend (not directly from frontend) for API key safety
